# Feature Specification: Perf Fixture Spec __NNN__

**Feature Branch**: `__NNN__-perf-fixture`

**Created**: 2026-05-28

**Status**: Draft

## Overview

Synthetic spec generated by `tests/perf/run.sh` to load-test the
reconciler. The shape mirrors a real `/speckit-tasks`-completed spec
(spec + plan + tasks with multiple phases) so the parser and
reconciler walk the same code paths they would for production specs.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Perf load (Priority: P1)

The reconciler walks this spec, parses each task phase, and emits the
same per-issue mutation set it would for a real spec.

**Acceptance Scenarios**:

1. **Given** spec + plan + tasks exist (no red-team / analyze
   markers, no checked tasks), **When** `src/reconcile.sh --dry-run`
   runs, **Then** the parser reports lifecycle phase `tasking` and
   four task phases per spec, and dry-run logs exactly one issue +
   four sub-issue mutations per spec.

## Success Criteria *(perf only)*

- **PC-001**: This fixture exists solely to populate the perf
  harness. No correctness assertions are intended; see the bats
  integration suite for those.
